# 크루 · 랜드마크 기획

> 상태: **기획 확정 / 미구현**  
> 목적: TourAPI 연동을 먼저 진행한 뒤, 크루·랜드마크 게임플레이 작업 시 이 문서를 기준으로 한다.  
> 최종 정리: 2026-07-23

---

## 1. 배경과 방향

### 현재 (구현됨)

- 개인이 깃발을 꽂고, 개인끼리 상대 깃발을 점령한다.
- 깃발 cost: 100 / 300 / 500 / 1000
- 반경: cost별 100 / 150 / 200 / 300m
- 점령 확률: 10 / 25 / 50 / 75%
- 전투력: 보유 active 깃발 cost 합
- 팀/크루 개념 없음

### 처음에 검토했다가 보류한 안

- 크루원이 같은 깃발에 포인트를 쌓아 키우고, 합동 점령(동참 알림·풀 모금·동시 재실)으로 뺏는 방식
- UX·실시간·어뷰징·지분 처리 부담이 커서 **1차로 채택하지 않음**

### 채택한 방향

1. **랜드마크(관광지 등)를 지도에 미리 둔다** — 쟁탈의 무대
2. 랜드마크 안에서는 **개인 깃발 꽂기·점령**만으로 싸운다 (합동 UX 없음)
3. **크루는 집계 단위** — 존 안 깃발 cost를 크루(또는 개인)별로 합산해 명칭 권리·Top10에 사용
4. 구현 순서: **TourAPI(랜드마크 데이터) → 랜드마크 존·깃발 규칙 → 크루·합산·표시**

---

## 2. 랜드마크

### 정의

- 지도에 미리 표시되는 고정 구역 (한국관광공사 TourAPI 동기화 및/또는 관리자 등록)
- 기본 영향 반경: **약 200m**
- 기존 `premium_places`(상점·쿠폰)와 **별도 엔티티**

### 표시

- 지도 마커 + 200m 범위
- 1위 주체가 있으면 이름 옆에 괄호 표기 (지하철역 스폰서 표기 느낌)
  - 크루 1위: 크루 프로필 이미지 + 크루명
  - 개인 1위: 닉네임
- 선택 시 정보 영역: 기본 정보 + **합산 전투력 Top10** (개인·크루 혼합)

### 운영 상태 (폐역)

- 랜드마크는 **삭제하지 않는다**
- 의미가 없어지면 **폐역(비활성)** 처리
- 폐역 시:
  - 상세 설명에 **「현재 운영되지 않는 랜드마크」** 안내
  - **깃발 꽂기·점령·집계·명칭·Top10은 그대로 유지** (규칙 엔진을 끄지 않음)
- 삭제·반경 복귀 기획은 없음

### 데이터 출처 (TourAPI 선행 작업)

- 한국관광공사 KorService2 등으로 후보 수집 후 DB 적재
- 관리자: 노출/폐역, 반경 오버라이드, 수동 추가
- 전국 전체 노출은 과다하므로 큐레이션·지역·필터는 TourAPI 작업 중 자연스럽게 결정

## 3. 랜드마크 안 깃발 규칙

| 항목 | 규칙 |
|------|------|
| cost | 기존과 동일 (100 / 300 / 500 / 1000) |
| 반경 | **항상 5m** (cost별 대형 반경 사용 안 함) |
| 개수 | 유저당 개수 제한 없음. **5m 충돌만** 준수하면 여러 개 가능 |
| 소속 판정 | 깃발 **중심 좌표**가 랜드마크 200m 안이면 해당 랜드마크 소속 |
| 외부 깃발과의 충돌 | 랜드마크 **밖** 일반 깃발의 큰 반경이 존 **안** 배치를 막지 않음 |
| 존 안 신규 | 항상 랜드마크형(5m) |
| 점령 | 기존 개인 점령 규칙과 동일 (확률·비용·방어 보상) |

### 랜드마크 생성 시 기존 깃발 편입

- 랜드마크가 **활성으로 생길 때**, 중심이 200m 안인 기존 깃발은 **반경을 5m로 줄여 편입**하고 합산에 포함한다
- 편입 후 5m 기준으로 드물게 겹치더라도 **기존 깃발은 유지** (신규만 5m 규칙 적용)
- 폐역·“삭제”로 반경을 다시 키우지 않음 → 복귀·겹침 폭발 문제 없음

### 현행 충돌 로직과의 관계

- 현재 `findActivePinsNear`는 기존·신규 반경 중 큰 값으로 충돌한다
- 랜드마크 존 안 배치는 **존 안 깃발끼리 5m 기준**으로만 충돌 검사하고, **존 밖 깃발 반경은 무시**해야 한다

## 4. 합산 · 명칭 권리 · Top10

### 집계 단위

- 대상: 해당 랜드마크 존 안 **active** 깃발
- 크루 소속 유저의 깃발 → **그 크루 버킷**에 cost 합산
- 미소속 → **개인 버킷** (닉네임으로 표시)
- Top10: **개인·크루 혼합** 주체 순위
- 반영: **실시간** (꽂기 / 점령 / 삭제 등 성공 직후)

### 명칭 (괄호) 1위

- 합산 전투력 1위 주체를 랜드마크명 옆에 표시
- **동점**이면 **해당 점수에 먼저 도달한 주체**
  - 정의: 그 주체의 합산이 현재 1위 점수에 **처음 도달한 시각**이 더 이른 쪽
  - 점수가 깨졌다가 다시 동점이 되면 **다시 도달한 시각** 기준

### 닉네임

- 개인 1위·Top10 개인 항목은 **닉네임** 표시
- 비속어는 **닉네임 설정 시** 차단 (괄호 표시 전제 과제)

### 크루 (랜드마크 단계에서의 역할)

- 이 단계의 크루는 “합동 점령 주체”가 아니라 **집계·표시 단위**
- 크루 생성/가입/프로필/하단 크루 탭 등은 **랜드마크·TourAPI 이후**에 구현
- 크루 미구현 기간에는 집계가 전부 개인 버킷이어도 규칙과 모순 없음

---

## 5. 크루 (후속 기획 요약)

TourAPI · 랜드마크 존·깃발·(가능하면) 개인 단위 합산/표시가 안정된 뒤 진행한다.

### 제품 스케치 (후속)

- 하단 **크루** 메뉴: 크루 목록 → 가입 → 소속 크루 현황 (인원, 멤버, 전투력, 크루 내 랭킹)
- 지도: **내 크루 깃발 하이라이트** 스위치
- 랜드마크 괄호·Top10에 크루 프로필·이름 반영
- 1인 1크루 등 운영 규칙, 가입/탈퇴/리더는 크루 구현 착수 시 상세 확정

### 명시적 비범위 (1차 크루에서도 하지 않기로 한 것)

- 크루원이 한 깃발에 포인트를 함께 쌓아 키우기
- 합동 점령 (개시·동참 알림·풀 모금·동시 재실)
- 크루 전용 깃발 소유로 개인 깃발을 대체하는 모델

---

## 6. 권장 구현 순서

1. **TourAPI** — 동기화, `landmarks`(가칭) 적재, 관리자 노출/폐역, 지도 표시
2. **랜드마크 존 깃발** — 5m, 충돌 예외, 생성 시 편입, 유저 다중 배치
3. **집계·명칭·Top10** — 우선 개인 버킷만으로도 가능
4. **크루** — 멤버십·프로필·집계 버킷·크루 탭·지도 토글
5. (장기) 필요 시 크루 심화 기능 — 본 문서 5절 비범위는 재기획 후
